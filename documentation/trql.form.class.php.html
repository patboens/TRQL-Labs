Class: Form Description: Properties:
Name Type Description
$self array Fixed 'class' information.
$wikidataId string Wikidata ID. Document with blank spaces/fields for insertion of required or requested information. A device for collecting data.
$aItems array Array of elements composing the form
$aAccepted array Array of inner tags that are accepted in a form
$szLang string Language of the form
$settings array Associative array with ALL settings used in the form:
- method (string) : HTTP method to be invoked to submit the form
- action (string) : Action fo the form (usually a URL or self to submit the form to the current script)
- tabindex (int) : tabindex in the form (use the $iTabIndex property of the current page ([doc]LSPage[/doc]) to hold a precise count between forms)
- withTabindex (bool) : Determines whether tabindex must be used or not. true by default
- mode (string) : Mode the form is entered. Can be 'edit' (modification of an existing record) or 'add' (add a new record). 'add' by default
- type (string) : Container type. 'form' by default
- version (string) : Form version. 'Web form 1.0' by default. Set to 'Web form 2.0' to benefit from HTML5 controls
- foreword (string) : Text that must precede the rendering of the form
- afterword (string) : Text that must follow the rendering of the form
- dataEnvironment (DataEnvironment) : A [doc]DataEnvironment[/doc] object. Warning: not really used at this stage (04-09-13 07:46)
- noECMA (bool) : Indicates whether we need to have the ECMA scripts be generated.false by default (= ECMA scripts generated)
- aECMA (array) : Internal array of scripts to be generated by Vae Soli! for entry checks and form validation
- withPreviousValues (bool) : Indicates if previous control values must be saved in the form
- preRenderFnc (string) : Callback to play before rendering of the form
- postRenderFnc (string) : Callback to play after rendering of the form
- noValidate (bool) : novalidate attribute of the form (HTML5)
- target (string) : Enumerated type (_blank, _self, _parent, _top). Specifies where to display the response that is received after submitting the form
- autocomplete (string) : Enumerated type (on, off). The off state indicates that by default, form controls in the form will have their resulting autocompletion state set to off; the on state indicates that by default, form controls in the form will have their resulting autocompletion state set to automatic.
- page (WebPage) : The page in which this form must be rendered. null by default
- hasFile (boolean) : Indicates whether the form is supposed to handle files (can be set manually; otherwise, it is treated automatically by examining all controls)
$szOnSubmit string Submit clause of the form
