https://creativecommons.org/licenses/by/4.0/legalcode Abstract: The Documentor class helps document ALL TRQL Labs classes defined in d:\websites\snippet-center\.

All TRQL classes share a common naming convention: trql..class.php. Because the Documentor is itself a TRQL Labs class, it documents itself. Because the Documentor is based on the most basic class, Thing, itself based onMother, the mother of all TRQL classes (the foundation of the foundation), we share a common behavior that is consistent across all the classes defined.

To document all the classes ― meaning … generating the documentation of each class ― there is little effort:



                                $o = new Documentor();

                                $o->documentAll();


There is no need to tell the Documentor class what must be documented. It goes without saying as it knows it!

Class: Documentor Description: Self-documentation class for the entirety of TRQL.
Class: DocumentorFile Description: A file
Class: DocumentorSourceFile Description: A source file. There is also a class called SoftwareSourceCode. We have decided not to use it so far and prefer to use the DocumentorFile class.
Class: DocumentorSourceFileObject Description: An object that is found in a DocumentorSourceFile: a history block, a class, a function, a parameter, a comment, a method, a property, a constant, a variable, ...
Class: DocumentorClass Description: A class
Class: DocumentorClassProperty Description: A property of a class
Class: DocumentorFileHeader Description: File header block
Class: DocumentorHistory Description: Change History block (can be pertaining to header, class, trait, interface, function or method)
Class: DocumentorFunction Description: A function or a method
Class: DocumentorFunctionParameter Description: A paramater of a function or method (both work the same way)
